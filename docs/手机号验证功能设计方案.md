# MindProject 手机号验证功能设计方案

## 一、需求概述

### 1.1 业务逻辑
- 用户注册仍使用**邮箱**（不变）
- 用户在论坛**发帖/回复/上传**等操作前，必须先**验证手机号**
- 未验证手机号的用户只能**浏览**，不能发布任何内容

### 1.2 技术方案
- 使用**阿里云号码认证服务**的短信验证码 API
- 验证码由阿里云生成、存储、校验，服务端不存验证码
- 两个核心接口：`SendSmsVerifyCode`（发送）、`CheckSmsVerifyCode`（核验）

## 二、数据库改动（MindAuth）

### 2.1 users 表新增字段

```sql
ALTER TABLE users 
  ADD COLUMN phone VARCHAR(20) DEFAULT NULL COMMENT '手机号',
  ADD COLUMN phone_verified TINYINT(1) DEFAULT 0 COMMENT '手机是否已验证',
  ADD INDEX idx_users_phone (phone);
```

### 2.2 不需要新建验证码表
阿里云负责验证码的生成、存储、过期管理，无需本地存储。

### 3.2 新增文件：src/utils/aliyunSms.js

```javascript
const crypto = require('crypto');

const ACCESS_KEY_ID = process.env.ALIYUN_ACCESS_KEY_ID;
const ACCESS_KEY_SECRET = process.env.ALIYUN_ACCESS_KEY_SECRET;
const ENDPOINT = 'dypnsapi.aliyuncs.com';

/**
 * 阿里云 RPC 签名
 */
function sign(params) {
  const sorted = Object.keys(params).sort().map(k => 
    `${encodeURIComponent(k)}=${encodeURIComponent(params[k])}`
  ).join('&');
  
  const stringToSign = `GET&${encodeURIComponent('/')}&${encodeURIComponent(sorted)}`;
  const hmac = crypto.createHmac('sha1', ACCESS_KEY_SECRET + '&');
  return hmac.update(stringToSign).digest('base64');
}

/**
 * 构建请求 URL
 */
function buildUrl(params) {
  const baseParams = {
    Format: 'JSON',
    Version: '2017-05-25',
    AccessKeyId: ACCESS_KEY_ID,
    SignatureMethod: 'HMAC-SHA1',
    SignatureVersion: '1.0',
    SignatureNonce: crypto.randomUUID(),
    Timestamp: new Date().toISOString().replace(/\.\d{3}Z/, 'Z'),
  };
  
  const allParams = { ...baseParams, ...params };
  allParams.Signature = sign(allParams);
  
  return `https://${ENDPOINT}/?${Object.keys(allParams).map(k => 
    `${encodeURIComponent(k)}=${encodeURIComponent(allParams[k])}`
  ).join('&')}`;
}

/**
 * 发送短信验证码
 * @param {string} phone - 手机号（11位）
 * @returns {Promise<{success: boolean, bizId?: string}>}
 */
async function sendSmsCode(phone) {
  const params = {
    Action: 'SendSmsVerifyCode',
    PhoneNumber: phone,
    SignName: '速通互联验证码',  // 阿里云赠送的签名，需在控制台确认
    TemplateCode: '100001',      // 阿里云赠送的模板，需在控制台确认
    TemplateParam: JSON.stringify({ code: '##code##', min: '5' }),
    CodeLength: 6,
    ValidTime: 300,    // 5分钟有效
    Interval: 60,      // 60秒发送间隔
    CodeType: 1,       // 纯数字
  };
  
  const url = buildUrl(params);
  const resp = await fetch(url);
  const data = await resp.json();
  
  if (data.Code !== 'OK') {
    throw new Error(data.Message || '短信发送失败');
  }
  
  return { success: true, bizId: data.Model?.BizId };
}

/**
 * 核验短信验证码
 * @param {string} phone - 手机号
 * @param {string} code - 用户输入的验证码
 * @returns {Promise<{success: boolean}>}
 */
async function checkSmsCode(phone, code) {
  const params = {
    Action: 'CheckSmsVerifyCode',
    PhoneNumber: phone,
    VerifyCode: code,
  };
  
  const url = buildUrl(params);
  const resp = await fetch(url);
  const data = await resp.json();
  
  return {
    success: data.Code === 'OK' && data.Model?.VerifyResult === 'PASS'
  };
}

module.exports = { sendSmsCode, checkSmsCode };
```

### 3.3 新增文件：src/routes/sms.js

```javascript
const express = require('express');
const router = express.Router();
const { pool } = require('../db');
const requireAuth = require('../middleware/requireAuth');
const { createRateLimiter } = require('../middleware/rateLimit');
const { sendSmsCode, checkSmsCode } = require('../utils/aliyunSms');

/**
 * POST /api/sms/send
 * 发送短信验证码（需要登录）
 * Body: { phone: string }
 */
router.post('/send', requireAuth, createRateLimiter('sms:send', 3, 300), async (req, res) => {
  try {
    const { phone } = req.body;
    
    // 校验手机号格式
    if (!/^1[3-9]\d{9}$/.test(phone)) {
      return res.status(400).json({ error: '手机号格式错误' });
    }
    
    // 检查手机号是否已被其他用户绑定
    const [existing] = await pool.query(
      'SELECT id FROM users WHERE phone = ? AND id != ?',
      [phone, req.user.id]
    );
    if (existing.length > 0) {
      return res.status(400).json({ error: '该手机号已被其他用户使用' });
    }
    
    // 发送短信
    await sendSmsCode(phone);
    
    res.json({ success: true, message: '验证码已发送' });
  } catch (err) {
    console.error('[SMS] 发送失败:', err);
    res.status(500).json({ error: err.message || '发送失败，请稍后重试' });
  }
});

/**
 * POST /api/sms/verify
 * 验证并绑定手机号（需要登录）
 * Body: { phone: string, code: string }
 */
router.post('/verify', requireAuth, async (req, res) => {
  try {
    const { phone, code } = req.body;
    
    if (!phone || !code) {
      return res.status(400).json({ error: '参数不完整' });
    }
    
    // 调用阿里云核验
    const result = await checkSmsCode(phone, code);
    
    if (!result.success) {
      return res.status(400).json({ error: '验证码错误或已过期' });
    }
    
    // 绑定手机号到用户
    await pool.query(
      'UPDATE users SET phone = ?, phone_verified = 1 WHERE id = ?',
      [phone, req.user.id]
    );
    
    res.json({ success: true, message: '手机号绑定成功' });
  } catch (err) {
    console.error('[SMS] 验证失败:', err);
    res.status(500).json({ error: '验证失败，请稍后重试' });
  }
});

module.exports = router;
```

## 四、MindFourm 后端改动

### 4.1 新增守卫：src/common/guards/phone-verified.guard.ts

```typescript
import { Injectable, CanActivate, ExecutionContext, ForbiddenException } from '@nestjs/common';

@Injectable()
export class PhoneVerifiedGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    const user = request.user;
    
    if (!user) {
      throw new ForbiddenException('请先登录');
    }
    
    // 从 JWT payload 或数据库查询获取 phone_verified
    // 注意：JWT payload 里需要有 phone_verified 字段，或者在这里查数据库
    if (!user.phone_verified) {
      throw new ForbiddenException('请先验证手机号后才能执行此操作');
    }
    
    return true;
  }
}
```

- `GET /posts/:id` - 帖子详情
- `GET /categories` - 分类列表

### 4.4 JWT Payload 调整

确保 JWT payload 中包含 `phone_verified` 字段，这样守卫可以直接从 `req.user` 读取，无需额外查数据库：

```typescript
// 在生成 JWT 的地方（可能在 MindAuth 或 MindFourm 的 auth 模块）
const payload = {
  sub: user.id,
  username: user.username,
  email: user.email,
  phone_verified: user.phone_verified || false,  // 添加这个
  role: user.role,
};
```

**注意**：用户绑定手机后，需要让旧的 JWT 失效或重新登录，否则 JWT 里的 `phone_verified` 还是 `false`。

## 五、MindFourm 前端改动

### 5.1 新增组件：src/components/PhoneVerifyModal.tsx

```tsx
'use client';
import { useState } from 'react';

interface PhoneVerifyModalProps {
  onClose: () => void;
  onSuccess: () => void;
}

export function PhoneVerifyModal({ onClose, onSuccess }: PhoneVerifyModalProps) {
  const [phone, setPhone] = useState('');
  const [code, setCode] = useState('');
  const [step, setStep] = useState<'phone' | 'code'>('phone');
  const [loading, setLoading] = useState(false);
  const [countdown, setCountdown] = useState(0);
  const [error, setError] = useState('');

  // 发送验证码
  const sendCode = async () => {
    setError('');
    
    if (!/^1[3-9]\d{9}$/.test(phone)) {
      setError('请输入正确的手机号');
      return;
    }
    
    setLoading(true);
    try {
      const res = await fetch('/api/auth/sms/send', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        credentials: 'include',
        body: JSON.stringify({ phone }),
      });
      
      const data = await res.json();
      
      if (!res.ok) {
        throw new Error(data.error || '发送失败');
      }
      
      setStep('code');
      setCountdown(60);
      
      // 倒计时
      const timer = setInterval(() => {
        setCountdown(prev => {
          if (prev <= 1) {
            clearInterval(timer);
            return 0;
          }
          return prev - 1;
        });
      }, 1000);
    } catch (err: any) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  // 验证并绑定
  const verifyCode = async () => {
    setError('');
    
    if (!/^\d{6}$/.test(code)) {
      setError('请输入6位数字验证码');
      return;
    }
    
    setLoading(true);
    try {
      const res = await fetch('/api/auth/sms/verify', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        credentials: 'include',
        body: JSON.stringify({ phone, code }),
      });
      
      const data = await res.json();
      
      if (!res.ok) {
        throw new Error(data.error || '验证失败');
      }
      
      onSuccess();
      onClose();
    } catch (err: any) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center z-50">
      <div className="bg-white dark:bg-gray-800 rounded-xl p-6 w-96 max-w-[90vw]">
        <h2 className="text-xl font-bold mb-2">验证手机号</h2>
        <p className="text-gray-500 text-sm mb-4">
          发帖、回复等操作需要先绑定手机号
        </p>
        
        {error && (
          <div className="bg-red-50 text-red-600 text-sm p-3 rounded-lg mb-4">
            {error}
          </div>
        )}
        
        {step === 'phone' ? (
          <>
            <input
              type="tel"
              placeholder="请输入手机号"
              value={phone}
              onChange={e => setPhone(e.target.value.replace(/\D/g, '').slice(0, 11))}
              className="w-full border dark:border-gray-600 rounded-lg px-4 py-2 mb-4 bg-white dark:bg-gray-700"
              maxLength={11}
            />
            <button
              onClick={sendCode}
              disabled={loading || phone.length !== 11}
              className="w-full bg-blue-600 hover:bg-blue-700 text-white py-2 rounded-lg disabled:opacity-50 disabled:cursor-not-allowed transition"
            >
              {loading ? '发送中...' : '发送验证码'}
            </button>
          </>
        ) : (
          <>
            <p className="text-sm text-gray-600 dark:text-gray-400 mb-2">
              验证码已发送至{' '}
              <span className="font-medium">
                {phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2')}
              </span>
            </p>
            <input
              type="text"
              inputMode="numeric"
              placeholder="请输入6位验证码"
              value={code}
              onChange={e => setCode(e.target.value.replace(/\D/g, '').slice(0, 6))}
              className="w-full border dark:border-gray-600 rounded-lg px-4 py-2 mb-2 bg-white dark:bg-gray-700"
              maxLength={6}
            />
            {countdown > 0 ? (
              <p className="text-sm text-gray-400 mb-4">
                {countdown}秒后可重新发送
              </p>
            ) : (
              <button 
                onClick={() => { setStep('phone'); setError(''); }} 
                className="text-sm text-blue-600 hover:text-blue-700 mb-4"
              >
                更换手机号
              </button>
            )}
            <button
              onClick={verifyCode}
              disabled={loading || code.length !== 6}
              className="w-full bg-blue-600 hover:bg-blue-700 text-white py-2 rounded-lg disabled:opacity-50 disabled:cursor-not-allowed transition"
            >
              {loading ? '验证中...' : '验证并绑定'}
            </button>
          </>
        )}
        
        <button 
          onClick={onClose} 
          className="w-full mt-3 text-gray-500 hover:text-gray-700 py-2 transition"
        >
          取消
        </button>
      </div>
    </div>
  );
}
```

### 5.2 在发帖页面使用弹窗

```tsx
'use client';
import { useState } from 'react';
import { PhoneVerifyModal } from '@/components/PhoneVerifyModal';

    // 检查是否已验证手机

      setShowPhoneVerify(true);
      return;
    }
    
    // 正常发帖流程...
  };

  return (
    <>
      <button onClick={handleCreatePost}>发帖</button>

        <PhoneVerifyModal
          onClose={() => setShowPhoneVerify(false)}
```

### 5.3 全局拦截（可选）

如果想在多个地方拦截未验证手机的用户，可以封装一个 hook：

```tsx
// hooks/usePhoneRequired.ts
import { useAuth } from './useAuth';
import { useState, useCallback } from 'react';
import { PhoneVerifyModal } from '@/components/PhoneVerifyModal';

export function usePhoneRequired() {
  const { user, refreshUser } = useAuth();
  const [showModal, setShowModal] = useState(false);

  const requirePhone = useCallback((callback: () => void) => {
    if (!user?.phone_verified) {
      setShowModal(true);
      return;
    }
    callback();
  }, [user]);

  const modal = showModal ? (
    <PhoneVerifyModal
      onClose={() => setShowModal(false)}
      onSuccess={() => {
        refreshUser();
        setShowModal(false);
      }}
    />
  ) : null;

  return { requirePhone, modal };
}

// 使用方式
const { requirePhone, modal } = usePhoneRequired();

<Button onClick={() => requirePhone(() => { /* 发帖逻辑 */ })}>
  发帖
</Button>
{modal}
```

## 七、阿里云配置步骤

1. **开通服务**
   - 登录阿里云控制台
   - 搜索「号码认证服务」并开通

2. **获取签名和模板**
   - 进入控制台 → 短信认证服务 → 赠送签名配置
   - 选择一个系统赠送签名（如「速通互联验证码」）
   - 进入赠送模板配置，选择一个验证码模板，记录 TemplateCode

3. **创建 AccessKey**
   - 进入 RAM 访问控制
   - 创建用户，勾选「OpenAPI 调用访问」
   - 授权策略：`AliyunDypnsFullAccess`
   - 记录 AccessKeyId 和 AccessKeySecret

4. **配置环境变量**
   - 将 AK/SK 填入 MindAuth 的 `.env` 文件

## 八、注意事项

1. **JWT 刷新问题**
   - 用户绑定手机后，JWT 里的 `phone_verified` 还是旧值
   - 解决方案：绑定成功后让前端重新登录，或后端主动使旧 token 失效

2. **手机号唯一性**
   - 一个手机号只能绑定一个账号
   - 发送验证码时需检查是否已被其他用户绑定

3. **频率限制**
   - 同一手机号 60 秒内只能发一次（阿里云控制）
   - 同一用户 5 分钟内最多发 3 次（rate limiter 控制）

4. **签名和模板**
   - 必须使用阿里云赠送的签名和模板，自定义签名需要审核
   - 签名和模板代码需要在代码中配置正确

5. **费用**
   - 短信发送按条计费（约 0.045 元/条）
   - 验证码核验免费
